# Self Study Studio Canva 风格产品功能说明设计

日期：2026-07-13

状态：已确认

## 1. 目标

将现有产品功能说明升级为一套以图为主、可导入 Canva 继续编辑的双版式产品资料：

1. 16:9 产品演示文稿，用于 5–10 分钟产品介绍和现场演示。
2. A4 纵向产品手册，用于完整阅读、归档和导出 PDF。

两套成品共享同一套产品事实、流程图、界面截图、状态判断和维护记录，不分别维护互相独立的内容副本。

## 2. 交付物

- `docs/product-guide/self-study-studio-product-deck.pptx`：16:9 可编辑演示稿，可导入 Canva。
- `docs/product-guide/self-study-studio-product-guide-a4.pdf`：A4 纵向阅读版，可导入 Canva。
- `docs/PRODUCT_GUIDE.md`：仓库内长期维护的产品功能说明权威入口。
- `docs/assets/product-guide/`：真实 App 截图、标注图和文档共用视觉资源。
- `diagrams/product-*.mmd|svg|png`：可维护的流程图源文件及静态输出。

当前环境没有直接写入 Canva 的连接器，因此不创建或覆盖用户的 Canva 云端项目。交付物必须可由 Canva 的 PowerPoint/PDF 导入能力继续排版和协作。

## 3. 内容架构

### 3.1 16:9 演示版

控制在 18–22 页，每页只承担一个主要结论：

1. 封面与一句话定位
2. 学习记录产品要解决的问题
3. 核心学习闭环
4. 产品原则与核心概念
5. 当前信息架构
6. 功能模块关系
7. 首次使用流程
8. Today 与继续学习入口
9. Quick Log 流程与界面演示
10. Timer 流程与界面演示
11. Proof 流程与界面演示
12. Project Trail 演示
13. Weekly Review 流程与界面演示
14. AI Review 及本地规则降级
15. Library 与数据导出
16. 标准 Demo 故事板
17. 功能状态矩阵
18. 当前限制
19. 已设计但未进入当前产品的能力
20. 总结与维护信息

根据截图可用性，可以把同一模块的流程图与界面演示拆成相邻两页，但总页数不超过 22 页。

### 3.2 A4 手册版

控制在约 25–35 页，沿用演示版的故事顺序，并补充：

- 用户目的、入口与前置条件
- 主流程、关键字段和系统规则
- 空状态、失败状态与降级方式
- 模块之间的数据关系
- Demo 操作、讲解词、预期结果和备用路径
- 功能实现状态、验证证据与已知限制
- 持续更新规则与按日期倒序的变更记录

### 3.3 仓库 Markdown 手册

`docs/PRODUCT_GUIDE.md` 采用与 A4 手册一致的章节结构。它是产品事实与实现状态的权威文本来源；PPTX 和 PDF 是视觉化发布版本。

## 4. 视觉系统

整体采用适合 Canva 二次编辑的现代编辑型风格：

- 暖白或浅中性色背景
- 深灰正文
- 蓝紫色作为主要强调色
- 绿色只表达完成或正常路径
- 橙色只表达提醒、等待或降级
- 大标题、宽留白、短段落
- 一页一个主要信息焦点
- 避免密集卡片墙和装饰性图表

流程图保持现有产品图的中性、清晰风格；界面截图使用统一设备画框和 3–5 个编号标注。颜色之外必须同时使用文字、箭头、线型或形状表达含义。

## 5. 图示策略

视觉内容占主要页面面积，优先级如下：

1. 真实 App 模拟器截图
2. 已核对的产品流程图
3. 功能关系图和信息架构图
4. 少量术语关系示意图

复用并核对现有 10 张产品图：

- 学习轨迹核心闭环
- 当前信息架构
- 功能模块关系
- Demo 故事板
- 首次使用流程
- Session 流程
- Proof 流程
- Weekly Review 流程
- AI Review 降级流程
- 数据导出流程

不为了增加图量重复表达同一关系。每张图必须紧邻对应正文，并附简短图注。

## 6. 演示截图

截图必须来自当前可运行 App，不用概念稿冒充已实现界面。目标截图包括：

1. Today 起点与 Continue 卡片
2. Quick Log 表单
3. Timer 进行中
4. Session 保存结果
5. Proof 添加入口
6. Proof Detail
7. Project Trail
8. Weekly Review
9. Library 浏览
10. Export 结果提示

截图使用统一的 CS336、吉他弹唱、DaVinci 调色示例数据。原始截图与带标注截图分别保存；不得暴露 API Key、个人文件名或其他敏感信息。

若某界面无法在当前构建中稳定复现，则在手册中明确标为限制，不生成伪造截图。

## 7. 内容准确性

判断优先级为：

1. 当前产品源码和可运行界面
2. 当前自动化测试
3. 当前构建与模拟器运行结果
4. README
5. 规格与实施计划

CloudKit/iCloud 同步、AI 课程规划和学习日历当前只作为“已设计”能力展示，不进入现行产品流程。

产品状态统一使用“已实现、部分实现、已设计、未规划”四级。所有完成性表述必须有代码、测试、构建或运行证据。

## 8. 双版式同步规则

- 产品事实先更新 `docs/PRODUCT_GUIDE.md`。
- 导航、规则或数据流变化时，同步更新对应 Mermaid 源文件并重新渲染 SVG/PNG。
- 界面明显变化时，重新采集对应模拟器截图。
- PPTX 与 A4 PDF 使用同一批图稿和同一份内容摘要重新生成。
- 每次发布记录核对日期、对应 Git commit、测试基线和已知限制。

## 9. 验收标准

- 演示版和手册版均可打开，且可导入 Canva。
- 16:9 演示版为 18–22 页，A4 手册版约 25–35 页。
- 主要用户流程均有流程图，主要 Demo 步骤均有真实界面图或明确的缺失说明。
- 无文字溢出、遮挡、图片拉伸、不可读标注或断开的流程连接。
- 所有当前功能描述与仓库实现一致。
- 所有规划能力均明确标为“已设计”，不冒充已实现。
- README 提供产品手册、演示稿和图示索引入口。
- 生成方式与维护规则可由后续提交重复执行。
